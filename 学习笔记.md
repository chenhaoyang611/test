# Git 学习笔记

## 一、Git 三层结构

| 层 | 大白话 | 位置 |
|----|--------|------|
| 工作区 | 你正在改的文件 | 项目文件夹本身 |
| 暂存区 | 待办清单，告诉 Git 哪些要存 | `.git/index` |
| 本地仓库 | Git 存档的所有历史版本 | `.git` 隐藏文件夹 |

```
工作区  ──git add──▶  暂存区  ──git commit──▶  本地仓库
```

## 二、基础命令

### git init —— 把文件夹变成仓库

```powershell
git init
```

> 在文件夹里创建 `.git` 隐藏目录，从此这个文件夹就归 Git 管了。只有 `git init` 过的文件夹才能用 add / commit / push。

### .gitignore —— 告诉 Git 别管哪些文件

有些文件不需要提交（临时文件、密码、依赖包等），写在 `.gitignore` 里就行：

```gitignore
.DS_Store          # Mac 系统垃圾
Thumbs.db          # Windows 缩略图
node_modules/      # 依赖包（太大了）
.env               # 环境变量（有密码）
```

> `.gitignore` 本身要提交到仓库，这样团队共用的规则一致。

### 日常操作

```powershell
git config user.email "邮箱"     # 配置身份（不加 --global 只对当前仓库生效）
git config user.name "用户名"

git add -A                      # 暂存所有变更
git commit -m "提交信息"         # 提交到本地仓库

git remote add origin <url>     # 关联远程仓库
git push -u origin main         # 推送，-u 设上游后下次直接 git push

git reset --hard <hash>         # 回退到指定提交，丢弃之后的所有更改
git log --oneline --graph --all # 可视化提交历史
```

## 三、分支

### 大白话

> 把代码复印一份，在复印件上随便折腾，不影响正式版。搞好了合回去，搞砸了删掉分支。

### 命令

| 命令 | 作用 |
|------|------|
| `git branch` | 查看所有本地分支 |
| `git branch 分支名` | 创建分支 |
| `git switch 分支名` | 切换分支 |
| `git switch -c 分支名` | 创建 + 切换（一步到位） |
| `git merge 分支名` | 合并到当前分支 |
| `git branch -d 分支名` | 删除分支 |

### 合并冲突

当两个分支**改了同一个文件的同一行**，Git 不知道用哪个，就会停下来让你决定：

```
<<<<<<< HEAD
菠萝                             ← 当前分支的内容
=======
凤梨                             ← 要合并过来的内容
>>>>>>> feature
```

你手动编辑文件，保留想要的那行，删掉 `<<<<` `====` `>>>>` 这些标记，然后：

```powershell
git add .
git commit -m "解决合并冲突"
```

### 回到历史——两种方式

| 方式 | 命令 | 比喻 | 合适场景 |
|------|------|------|---------|
| 创建分支 | `git switch -c 新分支 <hash>` | 在历史节点插面旗 | 基于旧版本继续开发 |
| 分离头指针 | `git switch --detach <hash>` | 时光机裸奔过去 | 临时翻看，看完就走 |

## 四、Cherry-pick

> merge 是整盘端走，cherry-pick 是挑着吃。

```powershell
git cherry-pick <commit-hash>   # 只摘某一个提交到当前分支
```

### 踩坑速记

| 问题 | 解决 |
|------|------|
| `Author identity unknown` | 配 `git config user.name/email` |
| `Permission denied`（.git 只读） | 在本地终端手动执行 |
| `Failed to connect to github.com port 443` | 开代理或检查网络 |
| `src refspec xxx does not match any` | 分支被删，`git branch 分支名 <hash>` 重建 |
| 合并冲突（同一行被两边改了） | 手动编辑文件 → `git add` → `git commit` |
