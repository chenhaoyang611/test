# Git 学习笔记

## 一、三层结构

```
工作区 ──git add──▶ 暂存区 ──git commit──▶ 本地仓库 ──git push──▶ 远程仓库
```

## 二、命令速查

| 命令 | 干啥 |
|------|------|
| `git init` | 文件夹变仓库（创建 `.git`） |
| `git config user.name/email` | 配身份 |
| `git add -A` | 暂存所有改动 |
| `git commit -m "xxx"` | 提交 |
| `git remote add origin <url>` | 关联 GitHub |
| `git push -u origin main` | 推送 |
| `git reset --hard <hash>` | 回退，丢弃之后的一切 |
| `git log --oneline --graph --all` | 可视化提交线 |

## 三、.gitignore

不归 Git 管的文件写进去：`node_modules/`、`.env`、`.DS_Store` 等。

## 四、分支

> 复印一份代码随便改，搞好了合回去，搞砸了删分支。

| 命令 | 作用 |
|------|------|
| `git branch` | 查看分支 |
| `git switch -c 分支名` | 创建+切换 |
| `git merge 分支名` | 合并到当前分支 |
| `git branch -d 分支名` | 删除 |

### 合并冲突

两边改了同一行 → Git 停下让你选 → 手动编辑删掉 `<<<<` `====` `>>>>` → `git add` → `git commit`

### 回到历史

| 方式 | 命令 | 场景 |
|------|------|------|
| 创建分支 | `git switch -c 新分支 <hash>` | 继续开发 |
| 分离头指针 | `git switch --detach <hash>` | 临时看看 |

## 五、Cherry-pick

> merge 整盘端走，cherry-pick 挑着吃。

```powershell
git cherry-pick <hash>
```

## 六、GitHub 协作

```
Fork（网页点）→ Clone 到本地 → 开分支改 → Push → PR（网页点）
```

**Fork** = 把别人的仓库复制到自己名下，不然没权限改。改完通过 PR 申请合回去。

## 七、踩坑

| 问题 | 解决 |
|------|------|
| `Author identity unknown` | `git config` 配身份 |
| `Permission denied` | 手动在终端执行 |
| `port 443 连不上` | 开代理 |
| 分支丢了 | `git branch 分支名 <hash>` 重建 |
| 合并冲突 | 手动编辑 → add → commit |

## 八、个人理解

| 概念 | 我理解 |
|------|--------|
| Git | 版本管理工具 |
| commit | 快照，随时回退 |
| 分支 | 平行时间线，兼具快照能力 |
| GitHub | 超级网盘 + 协作平台（PR 等） |
